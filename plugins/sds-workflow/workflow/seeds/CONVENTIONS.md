# CONVENTIONS — 비타협 규칙 (저장소 템플릿)

이 문서는 `/pick` 이 플랜 생성 시 자동 체크하는 **비타협 항목**만 담는다. 보편적 가이드라인은 `CLAUDE.md` 에 둔다.

플랜의 "구현 접근" 이 이 중 하나라도 위반하면 에이전트가 플랜에 플래그를 박고, 위반을 해소하지 않은 채 구현 단계로 넘어가지 않는다.

> 이 파일은 `/init` 이 생성한 **seed** 다. 팀이 자신의 저장소 현황에 맞춰 섹션을 채워넣는다. 각 섹션은 예시이며, 적용되지 않는 항목은 삭제한다.

## 1. 계층 의존 방향

- (예시) `src/views/**`, `src/components/**` 는 `@/axios/**` 를 직접 import 하지 않는다
- (예시) 순환 참조 금지
- (예시) 의존 방향: `View → Composable → (Query/Data/Store/Utils/Types)`

## 2. 책임 분리

- (예시) HTTP 호출 계층은 순수 호출만. 비즈니스 로직·가공 금지

## 3. 타입

- (예시) 자동 생성 타입 파일 수동 수정 금지

## 4. 커밋·브랜치

- 커밋 type 은 저장소 commitlint 설정에 따름
- 커밋 메시지: `{type}: {ISSUE_KEY}-XXXX {한국어 요약}`

## 5. i18n / 접근성

- (예시) 사용자 노출 문자열은 하드코딩 금지

## 6. 테스트

- (예시) 신규 비즈니스 로직은 테스트 추가

---

※ 이 섹션들은 저장소 상황에 맞게 취사선택·수정한다. `/pick` 은 여기 명시된 규칙만 체크한다.
