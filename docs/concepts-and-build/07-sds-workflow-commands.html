sds-workflow 9 커맨드

집기 → 출하 → 착륙 + 부수 도구

/init
저장소 부팅 — .team-workflow/ 스캐폴드 + remote 등록
/pick PROJ-XXXX
Jira 이슈 가져오기 — 컨텍스트·브랜치·플랜 자동
/ship
검증 → 커밋 → 푸시 → MR 등록 → Jira 코멘트
/land PROJ-XXXX
머지 후 Jira RESOLVE 전환 (브랜치 보존)
/where
현재 상태 감지 → 다음 액션 안내
/draft "설명"
신규 Jira 이슈 초안 — 5섹션 구조화 + 중복 탐지
/autopilot PROJ-XXXX
자율 운행 — pick → 구현 → ship 원샷. 다중 issue 시 worktree 병렬
/tune "피드백"
피드백을 워크플로우에 구조화해 반영
/update
플러그인 업데이트 안내
민감정보 (Jira 키, GitLab URL 등) 는 plugin 코드가 아니라 consumer repo 의 .team-workflow/workflow.yml 에 보관 → plugin 자체는 어디서나 사용 가능.
