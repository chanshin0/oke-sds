충돌 해결 — dev rebase

migration branch ↔ dev 사이의 "delete vs modify" 처리

문제
migration branch 가 plugin/sds-workflow/ 통째 삭제 vs dev 에 후속 sds-workflow commits (session-metrics, autopilot --ralph 폐기 등) 머지됨 → "delete vs modify" 충돌 다수
→
해결 (2 단계)
1) dev 의 후속 변경 13 파일을 okestro-sds 로 sync + re-genericize
2) migration branch 를 git reset --hard origin/dev 로 깔끔히 재기반 + 새 단일 commit + force-push
결과 (Before)
5 commits (migrate, settings, rename, drop bundles, weekly-report)
+ "merge migrate1 → migrate2 → migrate3 → ..."
→
결과 (After)
1 commit (chore: migrate sds-workflow plugin out)
dev HEAD 위에 단정한 squashed re-apply
rebase + 충돌 일일이 resolve 하는 것보다 reset + 재적용이 단순. migration 의 의도는 "delete" 단일 vector 라 squash 가 자연스러움.
