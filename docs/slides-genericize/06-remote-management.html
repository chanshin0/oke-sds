2.3 remote 관리 보강

install 시 remote-sds 등록을 정확히 — 3 layer 안전장치

1
Phase 2.0 신설 — origin URL 자동 감지
기존 clone 인 경우 git remote get-url origin 파싱 → base_url + project_path 추출 → AskUserQuestion 의 default 로. 사용자가 두 번 타이핑할 필요 없음
2
Phase 4.5 강화 — exit 10 분기 즉시 처리
remote-sds 가 다른 URL 가리키면 ensure-remote-sds.sh 가 exit 10 반환. init.md 가 즉시 AskUserQuestion 으로 (a) 교체 / (b) 유지 / (c) 중단 결정. 미결 상태로 종료 금지 → ship 단계 사고 방지
3
scripts/ensure-remote-sds.sh — URL 형식 검증
URL: https?:// 또는 git@ scheme 강제. project_path: 슬래시 1개 이상, 선두 슬래시·.git 접미 금지. fetch 실패 시 명확한 안내 (인증/네트워크 확인)
